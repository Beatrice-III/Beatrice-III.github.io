About Project

  • Project Goals

    Our objective for the Transport Next Generation (TNG): Communicator plugins project is to implement and test new communicators for the TNG system. Specifically, we aim to achieve the following deliverables:

    1. Identify and select three new communication protocols for the TNG system.
    2. Implement APIs for each of the selected protocols that allow the TNG system to interface with them.
    3. Develop protocol-specific logic that enables connections between peers over each new protocol.
    4. Test the new communicators using a combination of automated and manual methods to ensure that they are functioning correctly.
    5. Provide a user guide and developer guide that clearly explain how to use and extend the new communicators.

    To accomplish these objectives, we will perform a literature review to research various communication protocols, analyze the existing TNG architecture, and develop integrations for each of the selected protocols. We will also conduct thorough testing to identify any bugs or issues that arise during development. While developing and testing the new communicators, we will document our work and ensure that our code adheres to best practices, to facilitate future development or updates to the TNG system.

  • Implementation

    To solve the Transport Next Generation (TNG): Communicator plugins problem, we will follow these technical steps:

    Conduct a literature review to identify potential communication protocols to implement in the TNG system. Consider factors such as efficiency, security, and scalability.
    Analyze the existing TNG architecture to understand how communicators will fit into the larger framework. Develop a plan for integrating new communicators into the system.
    Choose three communication protocols to implement based on the results of the literature review and the needs of the TNG system. Research the specifics of each protocol to prepare for implementation.
    Create APIs for each protocol that allow the TNG system to interface with them. This will involve developing data structures that can hold the state of connections and implementing functions that operate on those data structures.
    Implement protocol-specific logic required to establish connections between peers. For example, if implementing Bluetooth as a communication protocol, this step would involve implementing the logic for discovering and pairing with nearby Bluetooth devices.
    Test the new communicators using automated and manual methods. Set up a test environment that simulates different network conditions (e.g. high latency, low bandwidth) to ensure the communicators can handle real-world scenarios.
    Identify any bugs or issues that arise during testing and fix them. This may require changes to the core communicators code or adjustments to the TNG system itself.
    Create documentation for each communicator that explains how it works and how to use it. The developer guide should include information on how to extend or modify the APIs to add additional protocols or customize existing ones.
    Conduct code reviews to ensure that the code is properly formatted, commented, and adheres to best practices.
    Submit the completed work, including documentation, code, and testing results.
    The development and testing of the communicators will require knowledge in programming languages such as C and experience with communication protocols such as HTTPS, QUIC, Bluetooth, or WiFi mesh. Additionally, we will use Linux as the primary development platform, and utilize automated testing tools and manual testing techniques to ensure a robust final product.

  • Timeline

    • First month:

      • Week 1: Conduct a literature review on various communication protocols to identify potential communicators that could be implemented in the TNG system. Consider factors such as efficiency, security, and scalability.
      • Week 2: Analyze the existing TNG architecture to understand how communicators will fit into the larger framework. Develop a plan for integrating new communicators into the system.
      • Week 3: Choose three communication protocols to implement based on the results of the literature review and the needs of the TNG system. Research the specifics of each protocol to prepare for implementation.
      • Week 4: Begin coding the new communicators. This will involve developing APIs that allow the TNG system to interface with each protocol, as well as implementing the protocol-specific logic required to establish connections between peers.

      Second month:

      • Week 1: Continue coding the new communicators. Focus on completing the core functionality of each API and making sure it integrates properly with the TNG system.
      • Week 2: Begin testing the communicators using a combination of automated and manual tests. Set up a test environment that simulates different network conditions (e.g. high latency, low bandwidth) to ensure the communicators can handle real-world scenarios.
      • Week 3: Identify any bugs or issues that arise during testing and fix them. This may require changes to the core communicators code or adjustments to the TNG system itself.
      • Week 4: Run additional tests to ensure that all issues have been fixed and that the communicators are functioning correctly. Once all tests have passed, create documentation for each communicator that explains how it works and how to use it.

      Third month:

      • Week 1: Begin preparing the code for submission by cleaning up the codebase, organizing files and directories, and removing unnecessary code or comments.
      • Week 2: Finish documentation by creating a user guide that explains how to use the TNG system with the new communicators, and a developer guide that describes how the APIs work and how to extend or modify them.
      • Week 3: Conduct code reviews to ensure that the code is properly formatted, commented, and adheres to best practices. Use this opportunity to identify any last-minute issues or bugs that need to be addressed.
      • Week 4: Submit the completed work, including documentation, code, and testing results. Make sure to include a summary of the project, highlighting the key features and benefits of the new communicators.
